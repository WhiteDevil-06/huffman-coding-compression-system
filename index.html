Key Insight

Huffman Coding assigns shorter codes to frequent characters and longer codes to rare ones. This is the Greedy property — by always merging the two smallest-frequency nodes, we minimize the total weighted path length (= total encoding bits), which is provably optimal for symbol-by-symbol encoding.

Why Prefix-Free Decoding Works

Huffman codes have the prefix-free property: no codeword is a prefix of another. This means we can decode a bitstream unambiguously by walking the tree bit-by-bit — going Left on 0, Right on 1 — and emitting the character whenever we reach a leaf. No lookahead or separators needed.

📚 Algorithm Theory

Understanding why Huffman Coding is a Greedy Algorithm and how its optimality is proven.

🧠

What is a Greedy Algorithm?

A greedy algorithm makes the locally optimal choice at each step, hoping to find a global optimum. Key properties:

  • Greedy Choice Property — A locally optimal choice leads to a globally optimal solution.
  • Optimal Substructure — An optimal solution contains optimal solutions to its subproblems.
Huffman Coding satisfies both — making it a textbook-perfect greedy algorithm.
⚡

Why Huffman is Greedy

The Greedy Choice in Huffman is:

"At each step, merge the two nodes with the minimum frequency."

This local decision (always pick the smallest two) is made n-1 times. Each decision is irrevocable — we never reconsider a merge. This greedy structure builds the tree from the bottom up, ensuring the rarest characters end up deepest in the tree (longest codes).

✅

Proof of Optimality (Exchange Argument)

We prove Huffman produces an optimal prefix-free code:

  1. In any optimal tree, the two symbols with lowest frequency must be siblings at the deepest level.
  2. Assume they are not — then swapping them with the actual deepest pair cannot increase the total cost (since lower-freq symbols get longer codes → cost doesn't increase).
  3. This shows the greedy choice (merge smallest two) is always safe — it leads to an optimal solution.
  4. By optimal substructure, after merging f₁+f₂, the reduced problem (with the merged node) also has an optimal solution, which Huffman finds recursively.
  5. ∴ Huffman Coding produces a minimum-cost prefix-free binary code. ∎
🔢

Algorithm Steps

  1. Count character frequencies → O(n)
  2. Insert all characters into a Min-Heap as leaf nodes → O(n)
  3. While heap has >1 node: extract two minimums, merge into internal node with combined frequency, re-insert → O(n log n)
  4. DFS the tree to assign binary codes → O(n)
  5. Encode original text using the code table → O(n)
🔑

Prefix-Free (Uniquely Decodable)

Huffman codes are prefix codes — no codeword is a prefix of another. This is guaranteed by the binary tree structure: codes are paths from root to leaf, and a leaf cannot be an ancestor of another leaf.

This means the encoded bitstream can be decoded unambiguously without any separator characters — just walk the tree bit by bit.
🌍

Real-World Applications

  • JPEG, PNG, MP3 — Huffman is the entropy coding stage
  • ZIP / DEFLATE — combines LZ77 + Huffman
  • HTTP/2 HPACK — header compression
  • Fax machines — Modified Huffman coding (MH)
  • Morse code — conceptually similar (frequent letters = shorter codes)

Compression Quality

Huffman achieves expected code length L* where:

  • Shannon Entropy H(X) Lower bound
  • Huffman L* H(X) ≤ L* < H(X)+1
  • Fixed-length (8-bit) 8 bits/char
Huffman is within 1 bit per symbol of the theoretical minimum (Shannon entropy). For grouped symbols, it approaches the entropy limit.
