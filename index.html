📚 Algorithm Theory

Understanding why Huffman Coding is a Greedy Algorithm, and how Shannon-Fano utilizes Divide & Conquer.

🧠

Greedy Strategy (Huffman)

A greedy algorithm makes the locally optimal choice at each step. In Huffman, the choice is to continually merge the two smallest frequency nodes from a Min-Heap. This bottom-up approach guarantees a globally optimal layout.

Provably optimal: achieves the mathematical minimum expected code length.
⚡

The Greedy Choice Property

The greedy choice in Huffman — always merging the two nodes with lowest frequency — is irrevocable and can be proven to lead to a globally optimal tree via the Exchange Argument.

If two rarest symbols are not siblings at the deepest level, swapping them reduces total cost — contradiction. ∴ greedy is safe.
✂️

Divide & Conquer (Shannon-Fano)

Top-down recursive heuristic: sort characters by frequency, then repeatedly split into two groups whose probability sums are as equal as possible. Assign 0 to the left group and 1 to the right.

Sub-optimal: equal splitting does not guarantee minimum expected code length.
⚖️

Why Shannon-Fano Can Fail

The balanced-split heuristic occasionally assigns a shorter code to a less frequent character and a longer code to a more frequent one — directly contradicting the optimal assignment. Huffman's bottom-up merge avoids this.

Shannon-Fano's expected length L satisfies H(X) ≤ L, but L may exceed Huffman's L* significantly for skewed distributions.
✅

Proof of Optimality (Huffman — Exchange Argument)

In any optimal tree, the two rarest symbols must be siblings at the deepest level. If not, swapping them can only reduce (or keep equal) total cost — contradiction. ∴ Huffman's greedy merge is always safe. ∎

🔢

Huffman Algorithm Steps

  1. Count character frequencies → O(n)
  2. Insert into a Min-Heap as leaf nodes → O(n)
  3. Repeat n-1 times: extract two minimums, merge, re-insert → O(n log n)
  4. DFS to assign binary codes → O(n)
  5. Encode text → O(m)
🔢

Shannon-Fano Algorithm Steps

  1. Count character frequencies → O(n)
  2. Sort characters by frequency (descending) → O(n log n)
  3. Find the split point that minimises frequency-sum difference → O(n)
  4. Assign 0 to left group, 1 to right group
  5. Recurse on each group until single symbols remain
🔑

Prefix-Free (Uniquely Decodable)

Both algorithms produce prefix codes — no codeword is prefix of another. This guarantees unambiguous decoding by walking the tree bit-by-bit without lookahead or separators.

Left on 0, Right on 1 — emit character at each leaf.
🌍

Real-World Applications of Huffman

  • JPEG, PNG, MP3 — entropy coding stage
  • ZIP / DEFLATE — combines LZ77 + Huffman
  • HTTP/2 HPACK — header compression
  • Fax machines — Modified Huffman (MH)
🌍

Real-World Contexts of Shannon-Fano

  • Historical compression — early telecom systems
  • Teaching tool — illustrates why greedy beats heuristics
  • ZIP (early) — some early variants used SF-style splitting
  • Information theory — foundational to Claude Shannon's 1948 paper
